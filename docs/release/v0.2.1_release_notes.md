## Scope

v0.2.1 is the **quality + accessibility** release that closes the v0.2-era
backlog identified by the 2026-05-23 three-role collaborative audit
(architect + scientific-advisor + dev-orchestrator) and lands the
first-ever user-facing materials for non-developer audiences:

1. **15 audit-fix items** from the collaborative-audit synthesis
   (commit `7b53bf2`) — 5 CRITICAL, 5 HIGH, 5 MEDIUM.
2. **First Edition user guide** (commit `3c8c546`) — 4365-line / 6.1 MB
   Markdown + PDF manual for first-time wet-lab users, with a UI integration
   that exposes the PDF from the upper-right of the workspace nav.
3. **README.md complete rewrite** (commit `3c8c546`) — 751 lines covering
   scope, architecture, dev philosophy, fabrication pathways, SnapGene
   collaboration, version history, and IP/licensing.
4. **Release-package zip + Python wheel** (this release) — self-contained
   download for research labs.

This release does NOT add new science: no new MR predicates beyond MR-59,
no new corpus records, no new ports, no new schemas. v0.2.0 already shipped
all of the v0.2 cadence scope. **v0.2.1 hardens the existing release** by
removing the "hollow scaffold" defects flagged by the audit (data-correctness
defects in the corpus; enforcement gaps in CI gates; runtime-binding gaps in
the composition root) and makes the system usable by junior researchers who
were never going to read ARCHITECTURE.md.

## Migration from v0.2.0 → v0.2.1

There is **no breaking change** from v0.2.0:

- The hosts schema v1.1 is backward-compatible (no change in v0.2.1).
- The markers catalogue is unchanged (no record additions/removals in v0.2.1).
- The corpus is unchanged (148 records; v0.2.1 fixed 2 records: EYFP +
  SV40 polyA, no count change).
- The Python public API is unchanged. `ValidationOrchestrator` accepts a
  new optional `markers_resolver` kwarg; omitting it preserves v0.1.0/v0.2.0
  baseline behaviour.
- The UI adds one icon; no existing behaviour removed.

For new callers wanting the v0.2.1 features:

- Wire `MarkersCataloguePort` into your `ValidationOrchestrator`:
  ```python
  from adapter.catalogue.yaml_markers_catalogue import YamlMarkersCatalogue
  from engine.markers_resolver import MarkersResolver
  from app.validation_orchestrator import ValidationOrchestrator, BiologyAdapterSet
  catalogue = YamlMarkersCatalogue(
      catalogue_path="catalogues/markers.yaml",
      schema_path="schemas/markers.schema.json",
  )
  resolver = MarkersResolver(port=catalogue)
  orchestrator = ValidationOrchestrator(
      biology_adapters=BiologyAdapterSet(),
      markers_resolver=resolver,
  )
  ```
- Promote your custom CI gates to enforced once you've observed a stable
  baseline (template: `ml_corpus_license_check.py::lifecycle_state`).
- Install the credential-scan pre-commit hook:
  ```bash
  python -m pre_commit install
  ```
