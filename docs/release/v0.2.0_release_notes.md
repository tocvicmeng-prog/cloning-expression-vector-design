# Cloning & Expression Vector Design Toolkit 0.2.0 Release Notes

Release date: 2026-05-23
Release documentation refreshed: 2026-05-23

## Scope

This release ships the **v0.2 Enrichment Amendment** (21/21 cadence task cards
across two new phases) plus the post-cadence corpus-completion backlog work.
v0.2 was scoped per the standing 10-step working-principle cadence
([[cev-workflow-discipline]]) with three skills (`/scientific-advisor`,
`/architect`, `/ip-auditor`) coordinated by `/dev-orchestrator`.

The release closes 13 commits ahead of v0.1.0, including a 91-record
seed of the ML training corpus that grew to 148 records via three
post-cadence backlog passes.

## Migration

There is no breaking change from v0.1.0. The hosts schema is backward-
compatible (additive); the markers catalogue is new (introduces no
removal); the ML training corpus subsystem is out-of-runtime and does
not affect the runtime engine. Existing v0.1.0 callers continue to
work unchanged.

For new callers wanting the v0.2 features:
- Wire `MarkersCataloguePort` (canonical port #51) into your composition
  root via `adapter.catalogue.yaml_markers_catalogue.YamlMarkersCatalogue`.
- Optionally pass a `MarkersResolver` to `CompatibilityChecker` and
  `ValidationContext` if you want forward-looking access to marker
  metadata (Phase 5/6 makes use of this).
- Read `catalogues/markers.yaml` for the new markers catalogue.
- Read `docs/ml_corpus/` for the ML training corpus subsystem.

## Safety

Generated outputs remain advisory. Screening, institutional authorisation,
signed SOP-template integrity, and export gates remain mandatory for
operational use. The UI exposes design and review state without bypassing
institutional review, vendor screening, or professional judgement.

The ML training corpus (`docs/ml_corpus/`) is training data, not runtime
configuration. The `import-linter` contract `ml-corpus-is-not-runtime`
enforces the boundary; the runtime engine (`src/*`) must not import from
`docs/*`. Curator-side workflows for adding new records follow the
`docs/handover/2026-05-23_corpus_completion_strategies.md` strategy
document (BR-16-compliant; manual-download + curator-processing workflow
for any Addgene-mediated sequence ingestion).

For any commercial fork, the Fork-Readiness Memorandum (`ARCHITECTURE.md
§ 9.6`) and the operational checklist (`docs/fork-readiness/checklist.md`)
are NORMATIVE. Counsel review is mandatory.
