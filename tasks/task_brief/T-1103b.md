# T-1103b Task Brief — Admin-service production implementation + IPC server + ACL + `ReviewQueueAdminPort` (v1.4 — production half split per B4-08; H4-01 review queue + M4-05 OS-level ACL tests)

## Scope

Replaced the `interface.admin_service` placeholder with the production
admin-service package and IPC client boundary:

- `src/interface/admin_service/auth.py` authenticates signed
  admin-principal tokens, binds them to domain principals, denies expired or
  untrusted credentials, and records security events.
- `src/interface/admin_service/handlers.py` serialises admin verb handling,
  allows only administrators or active developer-bootstrap principals on the
  expected bootstrap verbs, and wraps accepted replies with decision-record
  metadata.
- `src/interface/admin_service/ipc.py` implements the T-1103a length-prefixed
  JSON framing contract, an in-process serialized server shim, endpoint
  selection, and Windows/POSIX ACL metadata validation.
- `src/interface/admin_service/review_queue_admin.py` routes the
  `triage_review_queue_item` verb to `ReviewQueueAdminPort` through
  `ReviewQueueAdminResolutionService`.
- `src/adapter/ipc/admin_service_client_production.py` implements
  `AdminServiceClientPort` over an injectable transport and includes an
  in-process transport for integration tests.
- `docs/deployment/admin_service.md` documents Windows service and POSIX
  systemd deployment expectations.
