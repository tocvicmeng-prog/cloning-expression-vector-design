# T-312b Task Brief — Production `AuditKeyProvider` adapters + rotation service + offline verifier

## Scope

Implement the production half of the audit-key split from CODING_AGENDA § 2.3.16:

- File-backed audit-key escrow adapter with indefinite archive retention and no raw-key accessor.
- Windows-DPAPI and POSIX-keyring adapter facades with explicit CI/runtime fallback to the file escrow backend.
- Backend selector for `config.audit_key_backend = "file" | "windows_dpapi" | "posix_keyring"`.
- Admin/bootstrap-only rotation service that emits `AuditKeyRotated` with old/new versions, reason, and embedded signed `DecisionRecord` payload.
- Standalone offline verifier that reads `audit.sqlite` plus escrowed key archive and exits `0` for clean chains or `1` with failing row IDs.
- Security runbook for provisioning, rotation, compromise response, and recovery.
